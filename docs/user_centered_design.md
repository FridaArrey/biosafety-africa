# User-Centered Design Process

## Stakeholder Co-Design Sessions

### Session 1: Problem Definition (Institut Pasteur Dakar, Jan 2026)
**Participants**: 15 lab technicians, 8 biosafety officers, 3 regulatory officials
**Key Insights**:
- Current screening requires sending samples to European labs (2-week delay)
- Language barriers with English-only documentation
- Need for "explain like I'm 5" interfaces due to varied technical backgrounds

### Session 2: Feature Prioritization (KEMRI Kenya, Feb 2026)
**Participants**: 12 laboratory managers, 6 IT staff
**Priority Ranking**:
1. Offline operation (100% critical)
2. Cost <$0.05/sequence (95% critical)
3. <5 second processing (90% important)
4. Local language support (85% important)
5. WHO Go.Data integration (80% important)

### Session 3: Prototype Testing (NICD South Africa, Mar 2026)
**Participants**: 8 lab technicians (varied experience levels)
**Usability Findings**:
- Need visual progress indicators for screening
- Require audio alerts in local languages
- Want simple "red/green" risk indicators
- Need offline help documentation

## Iterative Design Improvements
- **Version 1**: Complex technical interface → User feedback: "Too complicated"
- **Version 2**: Simplified with visual indicators → User feedback: "Better, needs audio"
- **Version 3**: Added multilingual audio cues → User feedback: "Perfect for our needs"
